Input

So far, we used readLine() to get user input. The problem is that this will always be text, not numbers.

To get numbers, we do the following:

var input = readLine()!!.toInt()

You already know what var input = readLine() does. Let's break the rest apart:
!! - This is to tell the computer that we don't care about null safety. You don't have to understand that right now.
.toInt() - In Kotlin, there are different types of data. Here are some that you should know:

  • String - Plain Text
  • Integer (short form: int) - Whole Numbers, so without decimal places
  • Double - Numbers with decimal places.
  • Boolean - True or False

Unlike other programming languages, the data type will not change "just because".
To change it, we have to use methods like .toInt() to change the datatype. Be careful with this method, since it will throw an error when using it on non-number text.
