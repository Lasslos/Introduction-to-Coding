How this course works

This course is made for absolute beginners. There will be different types of lessons. There will be two types of tasks: Theory and Exercise

  • Theory: Instead of reading the description, it should be taught by an experienced programmer. Always code along! That means that you always should try out, how something shown to you works. Instead of asking what if questions, try it yourself and then ask or make a comment!
  • Edu: Coding exercise that is validated by tests. It starts with a custom function. These functions will get some information and return some information.

What are custom functions?

Almost every code in Kotlin executes in functions. This is a function:


    fun name(input: Int): String {

    }

Lets break that apart.
fun is a keyword. It tells the computer that this is a function.
name is the name of the function. It can be anything.
(input: Int) This is the section of the function that declares what the input will be. Every time the function is executed, some test functions will give you a set of values that you don't know. But how exactly does the input work?

input is just the name. It can be anything and will normally be something meaningful that makes sense in the context.
: Int is the type of data that will be in the variable. What exactly the different data types are is not important right now.

Last but not least, there is : String at the end of the function. This is, again, a type of data. Here, it is the type of data that the function result should be. This result is "returned" by: return "result"
